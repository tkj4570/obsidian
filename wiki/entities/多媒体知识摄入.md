# 多媒体知识摄入

将视频、音频、播客等非文本媒体内容转化为结构化知识，纳入 LLM Wiki 知识库的方法体系。

## 核心能力

### 1. 格式转换
- **视频→文字**：通过字幕提取、语音识别（ASR）将视频转为文字
- **音频→文字**：播客、会议录音等音频内容的文字化
- **图片→描述**：图像内容的语义理解和文字描述生成

### 2. 内容结构化
- **元数据提取**：标题、作者、时间、标签、章节划分
- **内容分段**：按时间轴或主题逻辑将内容分段
- **关键词识别**：自动识别领域术语和关键概念

### 3. 知识关联
- **自动标注**：为新内容自动添加相关标签和分类
- **链接生成**：建立与现有知识库的跨媒体关联
- **矛盾检测**：识别新内容与已有知识的潜在冲突

## 技术栈

| 类型 | 工具/方法 | 适用场景 |
|------|-----------|----------|
| 字幕提取 | Bilibili Web Clipper | B 站视频 |
| 语音识别 | Whisper/OpenAI Whisper | 通用音频 |
| 图像理解 | GPT-4V/Claude 3.5 | 图片内容 |
| API 接口 | Local REST API | Obsidian 集成 |

## 工作流程

```
多媒体内容 → 提取 → 结构化 → 关联 → 归档
     ↓
Raw层 → Processing → Wiki层 → Schema层
```

## 应用场景

- **学习资料**：课程视频 → 知识点卡片
- **会议记录**：视频/音频 → 会议纪要 + 行动项
- **创意内容**：灵感视频/音频 → 创意库
- **技能培训**：操作视频 → 步骤指南

## 质量控制

- **完整性检查**：确保关键信息不丢失
- **准确性验证**：对 AI 转换结果进行人工审核
- **时效性标注**：标记内容的时间维度和可能过时信息
